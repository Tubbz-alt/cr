36 	<p>The package {@code javafx.beans.property} defines read-only
37 		properties and writable properties, plus a number of implementations.
38 	</p>
39 	<h2>Read-only Properties</h2>
40 	<p>Read-only properties have two getters, {@code get()} returns the
41 		primitive value, {@code getValue()} returns the boxed value.</p>
42 	<p>It is possible to observe read-only properties for changes. They
43 		define methods to add and remove {@link
44 		javafx.beans.InvalidationListener InvalidationListeners} and {@link
45 		javafx.beans.value.ChangeListener ChangeListeners}.</p>
46 	<p>To get the context of a read-only property, two methods {@code
47 		getBean()} and {@code getName()} are defined. They return the
48 		containing bean and the name of a property.</p>
49 
50 	<h2>Writable Properties</h2>
51 	<p>In addition to the functionality defined for read-only
52 		properties, writable properties contain the following methods.</p>
53 	<p>A writable property defines two setters in addition to the
54 		getters defined for read-only properties. The setter {@code set()}
55 		takes a primitive value, the second setter {@code setValue()} takes
56 		the boxed value.</p>
57 	<p>All properties can be bound to {@link
58 		javafx.beans.value.ObservableValue ObservableValues} of the same type,
59 		which means that the property will always contain the same value as
60 		the bound {@code ObservableValue}. It is also possible to define a
61 		bidirectional binding between two properties, so that both properties
62 		always contain the same value. If one of the properties changes, the
63 		other one will be updated.</p>
